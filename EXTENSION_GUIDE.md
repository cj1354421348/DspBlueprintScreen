# 扩展建筑指南

本文档旨在说明如何在应用中添加对新建筑的支持，以便在界面上正确显示其名称和图标。

> **注意**：本指南由 AI 根据项目代码结构反向工程生成，并经过了多次严格修正。

## 核心原理

本应用被设计为通用的蓝图解析工具。它能自动从蓝图字符串中读取建筑的 `itemId`（物品ID）。

因此，要支持一个新的建筑，您**不需要**修改任何解析逻辑。您唯一需要做的，就是为新的 `itemId` 提供它对应的**元数据**（主要是名称和图标）。

## 核心文件

-   **类型定义**: `src/data/types.ts` (定义了 `Item` 接口的结构)
-   **数据文件**: `src/data/itemsData.ts` (包含了所有建筑和物品的列表)

## 添加步骤

1.  打开 `src/data/itemsData.ts` 文件。
2.  在文件底部的 `items` 数组中，为您的新建筑添加一个新的对象。

**示例：**

```typescript
{
    id: 9999,                          // ❗ 必须是游戏中新建筑的唯一物品 ID
    name: '新建筑的名称',                  // 用于在界面上显示的名称
    icon: 'new-building-icon',         // 对应的图标文件名 (不含扩展名)

    // --- 以下为当前未使用的预留字段 ---
    models: [],                        // (未使用) 预留给未来可能的3D模型功能
    grid: [/* page */ 3, /* row */ 1, /* column */ 1], // (未使用) 预留给未来可能的网格布局功能
    productive: true                   // (未使用) 预留给未来可能的增产剂相关功能
}
```

### 字段说明 (基于代码引用分析)

#### 核心字段 (当前版本正在使用)
-   `id: number`: **（最重要）** 必须与游戏内新建筑的物品 ID (`itemId`) 完全一致。这是将蓝图数据与显示信息关联起来的唯一钥匙。
-   `name: string`: 建筑的显示名称。
-   `icon: string`: 用于在界面上显示图标的文件名。

#### 预留字段 (当前版本未使用)
-   `models: number[]`: **未使用**。推测是为未来的3D模型功能预留。
-   `grid: [number, number, number]`: **未使用**。推测是为未来的网格布局功能预留。
-   `productive?: boolean`: **未使用**。此字段用于标记物品是否可被增产剂影响，供上游计算蓝图产量的应用使用。
